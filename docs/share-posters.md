# 分享图生成方案

## 结论

分享图的核心诉求是：中文清晰、数字准确、可稳定在本地生成，并能覆盖行情、交易、复盘、信息四类内容。

因此当前采用两层方案：

1. 本地 Canvas 模板直接生成 PNG，作为默认方案。
2. 开源/免费额度生图模型只作为可选背景或封面，不承载关键文字和交易数字。
3. 分享时允许用户选择背景来源：本地纹理、用户上传图片、NVIDIA AI 生成。

这样可以避免免费生图服务配额不稳定、中文文字渲染失真、行情数字被模型改写的问题。

## NVIDIA Build 模型备选

- Qwen-Image：中文和海报语义能力较强，适合作为高质量背景/封面模型。关键交易文字仍必须由本地模板绘制。
- qwen-image-2512：作为当前 NVIDIA AI 背景默认模型，用于 OpenAI-compatible 图像生成接口。
- flux.2-klein-4b：更轻量，适合快速生成抽象金融科技背景。
- FLUX.1-schnell：Apache 2.0，速度快，适合生成抽象金融背景和视觉氛围；当前保留直接模型端点兼容。
- Stable Diffusion 3.5 Medium：生态成熟，可用于自部署装饰图，但上线前需要按使用规模核对 Stability 许可。

## 接入边界

- 分享图里的标题、价格、PnL、日期、期权到期日、复盘结论必须由本地模板绘制。
- AI 生成内容只能进入背景层、插画层或可选封面层。
- 默认不依赖第三方生图 API，避免免费额度耗尽导致分享功能不可用。
- NVIDIA API Key 不在前端硬编码：可在设置页本地保存，或通过服务端 `NVIDIA_API_KEY` 环境变量提供。
- OpenAI-compatible 图像接口默认使用 `https://integrate.api.nvidia.com/v1`；如自托管 Qwen-Image NIM，可用 `NVIDIA_IMAGE_BASE_URL` 覆盖到自己的 `/v1` 地址。
- `/api/share-background` 只代理背景生成请求，返回 base64 图片给本地 Canvas 合成。

## 分享交互

用户点击任意“分享图”入口后，先选择背景来源：

- 选择本地背景：使用内置的“深海雷达 / 期权钟摆 / 冷静账本”等风格，完全离线可用。
- 使用我的图片：从本机选择图片作为背景，不上传到外部服务。
- NVIDIA AI 生成：选择 Qwen / FLUX 等模型，确认提示词后生成背景；生成失败自动回退本地背景。
- 直接生成默认分享图：跳过背景选择，沿用默认 Canvas 背景。

## 关于网页端自动化

可以写 Python + Playwright/Selenium 脚本打开 ChatGPT 或 Gemini 网页、粘贴 Prompt、等待图片生成并下载，但不建议作为 App 内能力：

- 网页端不是稳定接口，DOM、下载按钮、弹窗和登录态随时会变。
- 登录 Cookie、验证码、风控和限流会让自动化脚本经常失效。
- 这类脚本难以保证符合服务条款，也不适合部署给多用户使用。

更稳的折中方案是：系统生成“AI 背景 Prompt”并复制到剪贴板，用户手动粘贴到网页端生成背景图；核心文字和交易数据仍由本地模板绘制。

## 已覆盖模块

- 行情：生成市场行情快照。
- 交易：生成单笔交易执行分享图。
- 复盘/决策：生成决策或复盘结论分享图。
- 信息：生成情报材料摘要分享图。
- 设置：配置分享背景 Provider、默认 NVIDIA 模型和可选 NVIDIA API Key。
